How it works

No screenshots-and-guessing where it can help it. Pointer reads the labels macOS already hands to assistive tech; when an app has none, it falls back to looking at the screen.

  1. Reads the accessibility tree — the buttons, menus, and text macOS already exposes.

  2. A local model picks the answer — the control you meant, or reads the content you asked about. Offline, on your Mac.

  3. It highlights or tells you — you move your own hand and do it, which is how you remember it next time.

Read me before you open it

Two one-time setup steps

Early build shared with friends, so it skips the App Store polish. Five honest minutes and you're in.

  1. 1

    Let macOS open it. Not notarized yet, so you'll see an "unidentified developer" warning. Right-click Pointer → Open → Open. (Or run xattr -cr /Applications/Pointer.app once.)

  2. 2

    Give it a local brain. Pointer thinks with models that run on your Mac. Install Ollama, then run ollama pull qwen3:8b (and ollama pull qwen2.5vl:7b for the "look at the screen" vision feature). Keep Ollama running. The voice model is already bundled.

Requires macOS 14.2+ · Apple Silicon recommended · grant Accessibility (and Microphone / Screen Recording for voice & vision) when it asks.
